# Work Specs / เอกสารสั่งงานแยกตามแอป

โฟลเดอร์นี้ใช้สำหรับเอกสารสั่งงานปรับระบบ/เปิด PR ให้ Codex หรือ Agent อ่านโดยตรงจาก repo

ไม่ใช่ changelog และไม่ใช่เอกสาร lock กฎถาวรของระบบ

## โครงแยกตามแอป

```text
docs/work/
├── chincha-tea/   # งานปรับแอปร้านน้ำ ชินชา Tea POS
└── seafood-pos/   # งานปรับแอปกุ้ง โกอ้วน คลังซีฟู้ด
```

## กติกา

- งานของแอปชาให้เขียนใน `docs/work/chincha-tea/`
- งานของแอปกุ้งให้เขียนใน `docs/work/seafood-pos/`
- ห้ามเอา requirement ของสองแอปมาปนกันในไฟล์เดียว
- เอกสารแต่ละ PR ควรมี Goal / Scope / Acceptance Criteria / Do Not Touch
- Codex ควรอ่านไฟล์ spec นี้ก่อนเริ่มเขียนโค้ด
- หลังทำงานเสร็จ ถ้ามีการเปลี่ยนโครงสร้างใหญ่ ค่อยอัปเดต docs ถาวรหรือ changelog แยกต่างหาก
